## หลอนเกินจนเกิด Primitive Obsession

## Problem Statement
- ต้องการสร้าง **SDLC Project Document** ที่มีข้อมูลซับซ้อน (Requirements, Milestones)
- ใช้ Builder เข้ามาจัดการเรื่องการสร้าง object ที่ซับซ้อนเเทนการใช้ constructor ตรง ๆ
- **Clone Project** เดิมเพื่อสร้างเวอร์ชันใหม่โดยไม่กระทบกับต้นฉบับ เพื่อให้ได้ **Version** ที่ต่างกัน เเล้วใช้ **Builder** เเก้ไขข้อมูลบางส่วน เช่น เพิ่ม Requirement , Stakeholder , TeckStack,
---

## 🔑 Process ที่สำคัญที่ต้องรู้

### 1️⃣ **PROTOTYPE PATTERN - Deep Copy Logic**

#### **ทำไมต้องใช้ Prototype?**
- เมื่อต้องการ **สำเนาเอกสารโปรเจกต์ที่ซับซ้อน** โดยไม่ให้ objects กัน
- มี nested structures (Requirements, Milestones อยู่ในเอกสาร)

#### **Key Implementation Details:**

```typescript
// ❌ ปัญหา: Shallow Copy
const doc1 = new SDLCProjectDocument();
doc1.functionalRequirements = [req1, req2];
const doc2 = doc1;  // ชี้ไปตัวเดียวกัน!
doc2.functionalRequirements[0].description = "CHANGED";
// doc1.functionalRequirements[0] ก็เปลี่ยนด้วย 😱
```

```typescript
// ✅ แก้ไข: Deep Copy via clone()
public clone(): SDLCProjectDocument {
    const cloned = new SDLCProjectDocument();
    
    // 1. Simple values - copy by value
    cloned.name = `Copy of ${baseName}`;
    cloned.version = this._version;
    
    // 2. Arrays - spread operator to create new array
    cloned.stakeholders = [...this._stakeholders];
    cloned.techStack = [...this._techStack];
    
    // 3. Nested Objects - call clone() on each item
    cloned.functionalRequirements = this._functionalRequirements.map(req => req.clone());
    
    // 4. Objects with Date - create new Date instance
    this._milestones.forEach(m => cloned.addSingleMilestone(m.clone()));
    
    return cloned;
}
```

**Critical Points:**
- `Requirement.clone()` → returns **new Requirement** ✅
- `Milestone.clone()` → creates **new Date()** not reference ✅
- Getters return copies: `functionalRequirements: Requirement[]` → maps to clone ✅
- Setters also deep copy: `set functionalRequirements()` → maps incoming to clones ✅

### 2️⃣ **BUILDER PATTERN - Fluent Interface + Generic Type**

#### **ทำไมต้องใช้ Builder?**
- ต้องจัดการ **ขั้นตอนการสร้างออบเจกต์อย่างประมาณ** (separation of concerns)
- สามารถ **สร้าง format ต่าง ๆ** จากข้อมูลเดียวกัน (SDLCProjectDocument vs Markdown string)
- **Fluent API** ทำให้ code อ่านง่ายและสง่างาม

#### **Fluent Interface Pattern:**

```typescript
// Method chaining - ทุก method ต้อง return `this`
interface IBuilder<T> {
    reset(): this;          // 🔄 reset state
    setName(name): this;    // ⚙️ set property
    addRequirement(...): this; // ➕ add to array
    build(): T;             // 🎁 return final product
}
```

```typescript
// Usage - readable syntax
const builder = new SDLCBuilder();
builder
    .reset()
    .setName("MyProject")
    .setVersion("1.0.0")
    .addStakeholder("Developer")
    .addRequirement("FR1", "Do something", "High")
    .build();  // → SDLCProjectDocument
```

#### **Generic Type `T` - Different Output Formats:**

```typescript
// Builder ที่สร้าง SDLCProjectDocument
class SDLCBuilder implements IBuilder<SDLCProjectDocument> {
    private document: SDLCProjectDocument;
    build(): SDLCProjectDocument { ... }
}

// Builder ที่สร้าง Markdown string
class MarkdownReportSDLCBuilder implements IBuilder<string> {
    private markdown: string;
    build(): string { ... }
}

// Director รองรับทั้งสอง
const director1 = new SDLCDirector<SDLCProjectDocument>(sdlcBuilder);
const director2 = new SDLCDirector<string>(markdownBuilder);
```

**Key Implementation Details:**

| Builder                     | Accumulates                  | Returns               | Purpose                      |
| --------------------------- | ---------------------------- | --------------------- | ---------------------------- |
| `SDLCBuilder`               | `SDLCProjectDocument` object | `SDLCProjectDocument` | Create actual project object |
| `MarkdownReportSDLCBuilder` | String fragments             | `string` (Markdown)   | Generate report              |

### 3️⃣ **DIRECTOR PATTERN - Template Methods**

#### **ทำไมต้องใช้ Director?**
- **Encapsulate complex construction sequences**
- ให้ builder สร้าง object ตามขั้นตอนที่กำหนด
- สามารถเปลี่ยน builder โดยไม่ต้องแก้ Director logic

#### **Template Methods - 4 Project Archetypes:**

```typescript
class SDLCDirector<T> {
    // 1️⃣ AGILE: Loop-based sprints
    constructAgileSprints(sprintCount: number, startDate: Date) {
        this.builder.reset()
            .setName(`Agile Project (${sprintCount} Sprints)`)
            .setVersion("1.0.0")
            .setProblemStatement("Rapid iteration needed.")
            .addStakeholder("Product Owner")
            .addStakeholder("Scrum Master")
            // ...
        
        // 💡 KEY: Dynamic loop adds milestones
        for (let i = 1; i <= sprintCount; i++) {
            this.builder.addMilestone(`Sprint ${i}`, date, `Increment v0.${i}`);
        }
    }

    // 2️⃣ WATERFALL: Sequential phases
    constructStrictWaterfall(startDate: Date) {
        const date = new Date(startDate);
        
        this.builder.addMilestone("Requirement Freeze", date, "...");
        date.setDate(date.getDate() + 30);
        
        this.builder.addMilestone("Design Approval", date, "...");
        date.setDate(date.getDate() + 90);
        // ... (sequential phases)
    }

    // 3️⃣ STARTUP MVP: Quick 1-month timeline
    constructStartupMVP(appName: string) {
        this.builder.reset()
            .setName(`${appName} - MVP Launch`)
            .setVersion("0.1.0-alpha")
            // ... critical features only
    }

    // 4️⃣ DECOMMISSION: System shutdown process
    constructSystemDecommission(systemName: string) {
        this.builder.reset()
            .setName(`Decommissioning: ${systemName}`)
            // ... shutdown steps
    }
}
```

### 4️⃣ **POLYMORPHIC USAGE - Same Director, Different Builders**

#### **The Power of Generics:**

```typescript
// ✅ บันทึก XML, JSON, Markdown ทั้งหมดจาก Director เดียว
const agileDirector = new SDLCDirector<SDLCProjectDocument>(sdlcBuilder);
agileDirector.constructAgileSprints(4, new Date());
const projectObj = sdlcBuilder.build();  // → Object

const reportDirector = new SDLCDirector<string>(markdownBuilder);
reportDirector.constructAgileSprints(4, new Date());
const markdownText = markdownBuilder.build();  // → Markdown string
```

**Key Benefits:**
- **Code reuse**: Director logic ไม่ต้องเปลี่ยน
- **Flexibility**: เพิ่ม builder ใหม่ได้โดยไม่แก้ Director
- **Type safety**: `SDLCDirector<T>` รู้ว่า build() คืน type `T`

### 5️⃣ **CLIENT USAGE - Real-World Patterns**

```typescript
// Pattern 1: Director controls everything
const builder = new SDLCBuilder();

director.constructAgileSprints(4, new Date());
console.log(builder.build().toString());

// Pattern 2: Builder with Director for different format

    .reset()

    .setVersion("1.0.1-patch")
    .addRequirement("FIX-001", "Rollback", "Critical")
    .build();
```

## 🎯 Summary - Key Concepts

| Pattern              | Role                           | When to Use                                  |
| -------------------- | ------------------------------ | -------------------------------------------- |
| **Prototype**        | Deep copy of complex objects   | Clone existing document with nested data     |
| **Builder**          | Construct objects step-by-step | Build different formats from same data       |
| **Director**         | Orchestrate builder steps      | Encapsulate complex construction sequences   |
| **Fluent Interface** | Readable method chaining       | Make builder API elegant                     |
| **Generic Types**    | Type-safe polymorphism         | Support SDLCProjectDocument OR string output |

---

## 💡 Critical Implementation Rules

✅ **Deep Copy in Prototype:**
- Always return `new` instances for objects
- Use spread `[...]` for arrays
- Call `.clone()` recursively for nested objects
- Create fresh `new Date()` for Date fields

✅ **Fluent Interface in Builder:**
- Every method returns `this`
- `reset()` should clear all state
- `build()` returns the final product and resets state

✅ **Delegation in Director:**
- Director knows the **sequence** not the **details**
- Director calls builder methods in **specific order**
- Builder handles **accumulation** of data

✅ **Type Safety with Generics:**
- `IBuilder<T>` ensures build() returns correct type
- `SDLCDirector<T>` works with any builder implementing `IBuilder<T>`
